# 방파제: 교실용 v3

비밀 기여, 무임승차, 제한 공개, 공동체 생존을 다루는 교실용 실시간 사회 시뮬레이션입니다. 학생별 정확한 기여량은 공개하지 않고, 총합·성공/실패·기준 미달자 수·명예 보상 정도만 공개합니다.

## 실행

별도 빌드는 필요 없습니다. 방 만들기, 학생 참여, 비밀 기여 제출, 서버 판정을 쓰려면 `server.js`로 실행합니다.

```powershell
cd C:\Users\user\Documents\breakwater-game
node server.js
```

브라우저에서 `http://localhost:4173`을 열면 됩니다.

정적 화면만 확인할 때는 파이썬 서버도 쓸 수 있지만, 이 경우 방 코드 공유 기능은 동작하지 않습니다.

```powershell
cd C:\Users\user\Documents\breakwater-game
python -m http.server 4173
```

- 학생 화면: `http://localhost:4173/student.html`
- 교사 화면: `http://localhost:4173/teacher.html`

## 검사

```powershell
npm run check
npm test
```

`npm test`는 방 생성, 학생 입장, 라운드 시작, 비밀 기여 중복 제출, 서버 판정, 개인 기여량 비노출을 확인합니다.

## 배포

GitHub에 올린 뒤 Render에서 Web Service로 배포합니다.

1. GitHub에 새 저장소를 만듭니다.
2. 이 폴더의 파일 전체를 저장소 루트에 올립니다.
3. Render에서 New Web Service를 만들고 GitHub 저장소를 연결합니다.
4. Render 설정:
   - Runtime: Node
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Health Check Path: `/health`
5. `render.yaml`을 쓰는 Blueprint 배포를 선택해도 됩니다.

주의: 현재 방 정보는 서버 메모리에 저장됩니다. Render 인스턴스가 재시작되거나 새로 배포되면 열린 방은 초기화됩니다. 한 차시 수업 중 사용하는 용도에는 충분하지만, 장기 저장이 필요하면 데이터베이스를 붙여야 합니다.

## 수업용 규칙

- 6라운드 동안 항구 도시가 버티면 결산합니다.
- 학생은 매 라운드 자원 10개를 받고, 일부를 방파제에 비밀 기여합니다.
- 라운드마다 공개 폭풍, 숨은 파고 범위, 항구 기준 기여량이 제시됩니다.
- 방어 성공 조건은 모둠 총 기여량이 실제 폭풍 세기 이상인 것입니다.
- 공동 협력 성공 조건은 방어 성공과 함께 학생 80% 이상이 항구 기준 이상 기여하는 것입니다.
- 공동 협력 성공 시 방파제 강화 +1을 얻습니다.
- 방어 실패 시 도시 피해가 1칸 늘고, 피해 3칸이면 전원 공동 패배입니다.
- 단, 방어 실패 순간 방파제 강화를 2개 이상 보유하고 있으면 "강화 사용 투표"가 열립니다. 학생 과반이 찬성하면 강화 2개를 소모해 이번 피해를 막고, 부결되면 피해가 그대로 1칸 늘어납니다. 전원 투표 시 자동 판정되며, 교사가 "강화 투표 종료"로 직접 마감할 수도 있습니다. 누가 찬성/반대했는지는 공개되지 않고 투표 수만 표시됩니다.
- 남긴 자원은 개인 점수, 명예 보상은 +2점입니다.
- 최종 도시 피해가 0이면 전원 +6점, 피해 1이면 전원 +3점, 피해 2이면 보너스가 없습니다.
- 아슬아슬하게 실패하면 기준 미달자 중 1명이 항구 기록에 남을 수 있지만, 정확한 기여량은 공개하지 않습니다.

## 화면 구성

- 시작 화면에서 교사가 `방 열기`를 누르면 방 코드가 생성되고 교사 화면으로 이동합니다.
- 학생은 시작 화면에서 방 코드와 이름/모둠명을 입력하면 학생 화면으로 이동합니다.
- 학생 화면은 현재 라운드, 내 자원, 항구 기준, 비밀 기여 입력, 제한 공개 결과를 보여 줍니다.
- 교사 화면은 라운드 시작, 협상, 비밀 제출 열기, 제출 현황, 서버 판정, 피해/강화/기록을 관리합니다.
- 교사 화면도 기본적으로 학생별 정확한 기여량을 보지 않습니다.
- 오늘의 명예 보상은 방어 성공 시 기준 이상 기여자 중 최다 기여자에게만 표시됩니다.
