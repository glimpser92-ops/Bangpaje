# ⚓ 방파제 — 교실용 협력·배신 보드게임 (웹)

폭풍이 몰아치는 항구 도시. 모두가 자원을 모아 방파제를 쌓아야 도시가 살지만,
**내지 않고 남긴 자원은 전부 내 점수**가 됩니다.
도시가 무너지면(피해 3) **모둠 전원 패배**, 살아남으면 **점수 1등 한 명만 우승**.
'문턱 공공재 게임'을 교실에서 그대로 체험하는 수업용 게임입니다.

- 교사 화면(프로젝터용)과 학생 화면(태블릿·스마트폰)이 분리되어 있습니다.
- 교사가 방을 열면 **QR코드/주소/방 코드**가 표시되고 학생이 접속해 참여합니다.
- 학생 기여는 **익명**입니다. 합계와 항구 기준 미달자 수만 공개되고, 정확한 개인 기여량은 공개되지 않습니다.
- 라운드마다 **항구 기준 기여량**이 제시되며, 공동 협력·감사·명예 보상은 이 기준 이상인지로 판단합니다.
- 방어 성공 + 모둠원 80% 이상 기준 이상 기여 시 방파제 강화가 쌓이고, 강화 2개는 실패 시 투표로 사용할 수 있습니다.
- 생존 모둠은 최종 피해에 따라 전원 보정(+6/+3/0점)을 받습니다.
- 게임 도중 **튕겨도 자동 재접속**됩니다. 기기를 바꿔도 이름으로 이어서 할 수 있습니다.
- 서버가 꺼져도 진행 상황이 파일로 저장되어 다시 켜면 이어집니다.

## 1. 실행하기 (학교 컴퓨터에서)

필요한 것: **Node.js 16 이상** (https://nodejs.org 에서 LTS 설치) — 다른 설치는 필요 없습니다.

```bash
node server.js
```

실행하면 터미널에 이렇게 나옵니다.

```
교사용(이 컴퓨터):  http://localhost:3000/teacher.html
학생 접속 주소:     http://192.168.x.x:3000   ← 같은 와이파이에서 접속
```

1. 교사용 주소를 열어 방을 만들고, 그 화면을 프로젝터에 띄웁니다.
2. 학생들은 QR을 찍거나 학생 접속 주소를 입력하고, 4글자 방 코드로 들어옵니다.
3. **교사 컴퓨터와 학생 기기가 같은 와이파이에 있어야 합니다.**

포트를 바꾸려면: `PORT=8080 node server.js`

> 접속이 안 될 때: 학교 와이파이가 기기 간 통신을 막는 경우(AP 격리)가 있습니다.
> 이때는 아래의 인터넷 배포 방법을 쓰거나, 휴대폰 핫스팟에 함께 접속하면 됩니다.

## 2. 인터넷에 배포하기 (선택)

같은 와이파이가 아니어도 접속하게 하려면 무료 호스팅에 올릴 수 있습니다.

- **Render** (render.com): New → Web Service → 이 폴더를 GitHub에 올려 연결 →
  Build Command는 비워 두고, Start Command를 `node server.js`로 설정.
- **Railway / Glitch** 등도 같은 방식입니다. `PORT` 환경변수는 자동으로 적용됩니다.

배포하면 교사 화면의 QR이 자동으로 배포 주소를 가리킵니다.

## 3. 수업 운영 팁

- 권장: 모둠당 4~6명, 6라운드, 협상 90초. (설명 10분 + 플레이 25분 + 토론 10분)
- 교사 화면은 개인 점수·정확한 개인 기여량을 표시하지 않으므로 **프로젝터에 띄워도 안전**합니다.
- 실패 시 공개되는 정보는 기준 미달자 수입니다. 1~2개 차이의 아슬아슬한 실패에서는 기준 미달자 중 1명이 기록에 남을 수 있습니다.
- 게임 종료 화면에 토론(디브리핑) 질문이 함께 나옵니다.
- 같은 모둠으로 **"한 판 더"**를 강력 추천합니다. 첫 판의 배신과 함락 경험이
  두 번째 판의 협상을 바꾸는 과정이 가장 좋은 토론거리입니다.
- 첫 화면의 **게임 방법** 아코디언을 수업 도입에 그대로 띄워 설명해도 좋습니다.

## 4. 자주 묻는 것

- **학생이 튕겼어요** → 같은 기기면 첫 화면의 "하던 게임으로 돌아가기" 버튼,
  다른 기기면 방 코드 입력 후 자기 이름을 골라 "이어서 하기".
- **제출 안 한 학생이 있어요** → 판정 버튼에 미제출 인원이 표시되며, 미제출은 0개로 처리됩니다.
- **교사 화면을 실수로 닫았어요** → 같은 브라우저에서 teacher.html을 다시 열면 자동 복구됩니다.
- **방은 얼마나 유지되나요** → 마지막 활동 후 12시간 뒤 자동 정리됩니다.

## 5. 보안에 대한 메모

"이어서 하기"는 교실 상황을 전제로 **이름만으로** 재접속을 허용합니다(비밀번호 없음).
장난으로 친구 이름을 고르면 친구가 튕기므로, 시작 전에 한 번 안내해 주세요.
인터넷에 공개 배포할 경우에도 방 코드는 무작위 4글자라 외부인이 우연히 들어오기는 어렵지만,
수업 시간에만 방을 운영하는 것을 권합니다.

## 폴더 구성

```
server.js          게임 서버 (의존성 없음)
public/index.html  첫 화면 — 학생 참여 + 게임 방법 설명
public/teacher.html 교사 화면 — 방 만들기, QR, 진행 대시보드, 결과
public/student.html 학생 화면 — 폭풍/협상/비밀 기여/판정
public/style.css   공통 디자인
public/qrcode.js   QR코드 라이브러리 (qrcode-generator, MIT)
data/rooms.json    진행 상황 자동 저장 (자동 생성)
```
